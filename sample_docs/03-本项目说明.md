# 本项目：Agentic RAG Assistant

本项目是一个对**任意 markdown 知识库**做"自主检索问答"的 Agentic RAG 助手，可作为
工程能力展示。你现在看到的这个在线 demo，语料就是这几篇公开示例文档。

## 技术栈
- **检索/编排**：LlamaIndex（`VectorStoreIndex` + workflow `ReActAgent`）。agent 自主
  决定是否调用知识库检索工具，再综合作答并给出处。
- **生成**：豆包 / 火山方舟（OpenAI 兼容接口，换模型只改环境变量，不锁死厂商）。
- **向量化**：本地 `bge-small-zh`（中文友好、体积小）。
- **服务**：FastAPI（HTTP 接口）+ Streamlit（本网页）+ Docker。

## 为什么不是"朴素 RAG"
朴素 RAG 把"检索→塞进 prompt→生成"写死。这里用 ReAct agent 让模型自主决策检索时机与
方式，更贴近真实助手，也更能体现 agent 编排能力。

## 实测指标
- 检索命中 hit@4：100%（人工标注小评测集）
- 检索延迟：P50 9ms / P95 27ms（本地 embedding + 内存索引）
- 每次作答都返回 top-k 源文件 + 相似度，可追溯

## 如何复用到自己的知识库
把语料目录换成任意 markdown 文件夹（改一个环境变量 `CORPUS_DIR`），重建索引即可。
源码与工程决策记录见 GitHub 仓库的 README 与 DECISIONS.md。
