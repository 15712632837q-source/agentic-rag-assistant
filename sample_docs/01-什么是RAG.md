# 什么是 RAG（检索增强生成）

RAG（Retrieval-Augmented Generation，检索增强生成）是一种让大语言模型在回答前先
"查资料"的方法。大模型本身的知识停在训练截止时间、且容易在细节上"一本正经地胡说"
（幻觉）。RAG 在生成答案前，先从一个外部知识库里检索出与问题相关的片段，再把这些
片段连同问题一起交给模型，让它"看着材料回答"。

## 基本流程
1. **切块（chunking）**：把文档切成若干段落级的小块。
2. **向量化（embedding）**：用 embedding 模型把每个块编码成向量，存进向量库。
3. **检索（retrieval）**：把用户问题也向量化，在向量库里找最相近的 top-k 个块。
4. **生成（generation）**：把检索到的块作为上下文，连同问题交给大模型生成答案。

## RAG 的价值
- **降低幻觉**：答案有外部材料支撑，可以给出"出处"。
- **知识可更新**：换知识库即可更新模型"知道"的内容，无需重训。
- **私域问答**：让模型回答它训练时没见过的私有/专有文档。

## 朴素 RAG 的局限
朴素 RAG 把"每问必检索一次再生成"写死。但有些问题不需要检索（闲聊、常识），有些
问题需要多轮检索或换个说法再查。固定管线无法自适应，这正是 **Agentic RAG** 要解决的。

## 衡量 RAG 质量的常见指标
- **检索命中率 hit@k**：正确的源文档是否出现在 top-k 检索结果里。
- **groundedness**：答案是否真的有检索到的材料支撑（而非模型编的）。
- **延迟 / 成本**：单次问答的耗时与 token 花费。
