# Agent 与 ReAct 模式

## 什么是 Agent
大模型本身只能输出文本，无法感知或改变外界环境。给它接上**工具**（读写文件、搜索、
调用 API、检索知识库等），让它能自己查信息、自己执行操作，这样的"大模型 + 一组工具 +
驱动循环"就构成了一个 **Agent**（智能体）。工具是大模型的"感官和四肢"。

## ReAct 模式
ReAct（Reasoning + Acting，推理与行动）是最常用的 Agent 运行模式。核心是一个循环：
- **Thought（思考）**：模型先想这一步要做什么、需不需要用工具。
- **Action（行动）**：调用某个工具（如检索知识库），并给出参数。
- **Observation（观察）**：拿到工具返回的结果。
- 重复 Thought → Action → Observation，直到信息足够，输出 **Final Answer（最终答案）**。

关键认知：让模型按 ReAct 一步步走，**主要靠系统提示词驱动**，而不是模型被专门训练成
这样。系统提示词就像给模型的"剧本"。另外，模型本身不能真正执行工具，它只是"请求"
调用，真正执行的是 Agent 的工具调用组件。

## Agentic RAG = RAG + Agent
把 RAG 的"检索"做成 Agent 的一个工具，让 ReAct agent **自主决定是否检索、检索什么、
要不要追问或多查几次**，就是 Agentic RAG。相比"每问必检索一次"的朴素 RAG，它更接近
真实助手的行为：简单问题可不检索直接答，复杂问题可多轮检索。代价是多一两次模型调用
（延迟和成本略增），通常用指标来权衡。

## Plan-and-Execute：另一种模式
除 ReAct 外，还有"先规划再执行"的模式（如 LangChain 的 Plan-and-Execute）：先由一个
Plan 模型列出步骤，执行 Agent 逐步执行，再由 Re-Plan 模型根据结果动态修改计划，循环
直到完成。它更适合步骤多、需要动态调整的复杂任务。
